# Titel "Aufgabe, was es Hauptsächlich beschreibt" (Beispiel: Camera Abstraction to Camera.cs)

## Beschreibung

- Bechreibung für das Problem / Aufgabe

- Was / Wie / Warum soll gemacht werden

- Was ist das Ziel

## Sub Issues

// Unteraufgaben, die erledigt werden müssen, um das Hauptissue zu erfüllen
// Die Hauptaufgabe beschreibt das Ziel und die Sub Issues sind die kleinen Schritte, um das Ziel zu erreichen
// Beispiel: Wenn das Hauptissue "Camera Abstraction to Camera.cs" ist, dann könnten die Sub Issues sein:
